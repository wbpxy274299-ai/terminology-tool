# 阿里云服务器部署指南

## 部署步骤

### 1. 上传文件到服务器

```bash
# 在服务器上执行（先 SSH 登录）
cd /www/wwwroot
# 把 terminology-tool.zip 上传到这个目录（用 FTP 或 scp）
```

### 2. 解压文件

```bash
cd /www/wwwroot
unzip terminology-tool.zip -d terminology-tool
cd terminology-tool
```

### 3. 安装依赖（如果需要）

```bash
# 这个项目不需要额外安装依赖，Node.js 自带 http/https/fs/path 模块
# 但需要确保服务器上有 Node.js
node -v  # 检查 Node.js 版本，建议 14+
```

### 4. 启动服务

```bash
# 先测试一下能不能正常启动
node server.js

# 看到这样的输出就成功了：
# ========================================
#   术语校对工具 - 服务器已启动
#   端口: 3000
#   访问: http://你的服务器IP:3000
#   AI中转: POST /api/ai
#   健康检查: GET /api/health
# ========================================
```

### 5. 后台运行（推荐）

```bash
# 安装 pm2（进程管理器，让服务一直在后台跑）
npm install -g pm2

# 用 pm2 启动
pm2 start server.js --name terminology-tool

# 查看状态
pm2 status

# 设置开机自启
pm2 startup
pm2 save
```

### 6. 配置域名（可选）

如果想用域名访问（比如 test-posting.xyz），需要在宝塔面板里配置反向代理：

1. 登录宝塔面板
2. 网站 → 添加站点 → 输入域名 test-posting.xyz
3. 设置 → 反向代理 → 添加反向代理
   - 代理名称：terminology
   - 目标URL：http://127.0.0.1:3000
4. 保存

### 7. 测试

- 访问 http://服务器IP:3000 （或 http://test-posting.xyz）
- 打开"贴文助手"
- 输入素材，点击"生成贴文"
- 如果能正常生成，说明部署成功！

## 常用命令

```bash
# 查看日志
pm2 logs terminology-tool

# 重启服务
pm2 restart terminology-tool

# 停止服务
pm2 stop terminology-tool

# 更新代码后重启
pm2 reload terminology-tool
```

## 故障排查

### 1. 端口被占用

```bash
# 查看 3000 端口被谁占用
netstat -tlnp | grep 3000

# 或者改 server.js 里的 PORT（第 12 行）
```

### 2. AI 接口超时

检查服务器能否访问 idealab.alibaba-inc.com：

```bash
curl -I https://idealab.alibaba-inc.com
```

### 3. 前端页面打不开

- 检查防火墙是否开放了 3000 端口
- 宝塔面板 → 安全 → 放行 3000 端口

## 架构说明

```
浏览器（用户）
    ↓ 访问 http://服务器IP:3000
服务器（你的阿里云）
    ↓ Node.js 托管静态网站
    ↓ 前端调用 /api/ai
    ↓ 服务器代理转发
idealab.alibaba-inc.com（AI 接口）
```

这样浏览器就不直接调 AI 接口了，通过你的服务器中转，完美解决 CORS 问题！
