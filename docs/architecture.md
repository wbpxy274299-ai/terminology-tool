# 架构说明

## 设计原则

1. **模块化**：每个功能独立成文件，职责单一
2. **数据分离**：6万条术语数据独立存储为 JSON，不嵌入 HTML
3. **按需加载**：数据异步加载，页面先展示，数据后台加载
4. **易于维护**：代码、样式、数据分开管理，改一处不影响其他

## 模块职责

### config.js
- 语言列表、名称、emoji、文件映射
- 全局常量，其他模块引用

### utils.js
- 文本标准化（全角转半角）
- HTML 转义（防 XSS）
- 关键词高亮
- Excel 文件读取
- 语言检测

### data.js
- TERMS 数组管理
- ALL_INDEX 索引构建
- 数据异步加载（fetch terms.json）
- 加载状态显示

### search.js
- 精确匹配 + 模糊匹配
- 搜索结果渲染
- 高亮关键词

### check.js
- 单条校对：长术语优先匹配
- 批量校对：逐行检查
- 结果渲染：高亮 + 对照表

### update.js
- 批量导入：多文件识别、合并、去重
- 单语言更新：增量更新、统计
- 更新后导出新 JSON

### app.js
- 事件绑定（Tab 切换、搜索、上传）
- 初始化流程

## 数据格式

```json
[
  ["中文", "日语", "英语", "韩语", "繁中", "越南语", "印尼语", "泰语", "分类", "key", "备注"],
  ["刺客", "アサシン", "Assassin", "암살자", "刺客", "Thích Khách", "Assassin", "แอสซาสซิน", "NPC挑战", "", ""]
]
```

- 数组索引 0-7：7种语言翻译
- 索引 8：分类（如"NPC挑战"）
- 索引 9：key（可选）
- 索引 10：备注（可选）
- 尾部空字段会被省略（压缩体积）

## 性能优化

1. **数据压缩**：去掉尾部空字段，从 15MB 降到 13.5MB
2. **异步加载**：页面先渲染，数据后台 fetch
3. **索引预构建**：加载时建立 ALL_INDEX，搜索时 O(1) 查找
4. **长术语优先**：校按时按长度排序，避免短术语误匹配

## 扩展建议

1. **添加新语言**：在 config.js 的 LANG_KEYS 中添加新语言代码
2. **添加新功能**：新建 src/js/feature.js，在 index.html 中引入
3. **换数据源**：修改 data.js 的 loadData() 函数即可
4. **添加后端**：如需团队协作，可加 Node.js 后端提供 API
